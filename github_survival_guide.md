# 🛟 Git ও GitHub - ডেভেলপারদের লাইফ-সেভার রুলবুক

টিমে কাজ করতে গেলে গিট (Git) নিয়ে ভুল হবেই, এটাই স্বাভাবিক। কিন্তু একজন "প্রো-ডেভেলপার" জানেন কীভাবে ম্যাজিকের মতো সেই ভুলগুলো ফিক্স করতে হয়। 

নিচে ডেভেলপারদের সবচেয়ে বেশি হওয়া ৬টি গিট মিস্টেক এবং তার সবচেয়ে সহজ সমাধান দেওয়া হলো:

---

### 🚨 ভুল ১: কমিট করার পর দেখলাম মেসেজে বানান ভুল বা ভুল নাম লিখেছি!
**সমস্যা:** তুমি `git commit -m "added backend routing"` দিতে গিয়ে লিখে ফেলেছ `"added backend rotoing"`। কিন্তু এখনো `git push` করোনি।
**সহজ সমাধান:** তোমার লাস্ট কমিটের নাম রিনেম (Rename) করো।
```bash
git commit --amend -m "added backend routing"
```
> [!TIP]
> `amend` মানে হলো সংশোধন করা। এই কমান্ড দিলে নতুন কোনো কমিট তৈরি হবে না, আগের কমিটের মেসেজটাই জাস্ট এডিট হয়ে যাবে।

---

### 🚨 ভুল ২: কমিট করে ফেলছি, কিন্তু একটা ফাইল Add করতে ভুলে গেছি!
**সমস্যা:** তুমি ৩টি ফাইল অ্যাড করে কমিট দিয়েছ, পরে দেখলে আরে! ৪ নম্বর ফাইলটাও তো এই কমিটেই যাওয়ার কথা ছিল।
**সহজ সমাধান:** ফাইলটাকে আগের কমিটের ভেতরেই ঢুকিয়ে দাও।
```bash
git add file4_name.js
git commit --amend --no-edit
```
> [!TIP]
> `--no-edit` মানে হলো, "আগের কমিট মেসেজটাই রাখো, শুধু আমি মাত্র যে ফাইলটা add করলাম সেটাকেও আগের প্যাকেজে ঢুকিয়ে নাও।"

---

### 🚨 ভুল ৩: ভুল ব্রাঞ্চে কাজ করে ফেলেছি! (খুব কমন)
**সমস্যা:** তুমি `main` ব্রাঞ্চে ছিলে, কিন্তু খেয়াল না করে সেখানে নতুন ফিচার লিখে ফেলেছ। এখন মনে পড়েছে কাজটা তো `feature` ব্রাঞ্চে করার কথা ছিল! এখনো কমিট করোনি।
**সহজ সমাধান:** কাজগুলো সাথে করে নতুন ব্রাঞ্চে নিয়ে যাও।
```bash
git stash
git checkout -b feature-branch
git stash pop
```
> [!TIP]
> `git stash` হলো একটা জাদুর বাক্স। তুমি যখন এটা দাও, কাজগুলো সেভ হয়ে বাক্সে ঢুকে যায় এবং তোমার ফাইল আগের মত ক্লিন হয়ে যায়। তারপর সঠিক ব্রাঞ্চে গিয়ে `git stash pop` দিলে বাক্স থেকে কাজগুলো আবার ফেরত আসে!

---

### 🚨 ভুল ৪: কোড করতে করতে সব গুবলেট পাকিয়ে ফেলেছি, নতুন করে শুরু করতে চাই!
**সমস্যা:** তুমি অনেকগুলো ফাইলে চেঞ্জ করেছ, কিন্তু কাজ করছে না। এখন তুমি চাও এই সব চেঞ্জ ডিলিট করে আধা ঘণ্টা আগের যে ক্লিন কোড ছিল সেখানে ফিরে যেতে।
**সহজ সমাধান:** পিসির সব আন-সেভড চেঞ্জ মুছে ফেলো।
```bash
git reset --hard HEAD
```
> [!CAUTION]
> এই কমান্ড দিলে তোমার সেভ না করা (uncommitted) সব কাজ ডিলিট হয়ে যাবে। তাই এটা দেওয়ার আগে ১০০ বার ভেবে নেবে যে তুমি সত্যিই সব ডিলিট করতে চাও কি না।

---

### 🚨 ভুল ৫: "Merge Conflict" এসেছে, আমি ভয়ে কাঁপছি!
**সমস্যা:** তুমি `git merge` দিয়েছ, কিন্তু টার্মিনালে লাল অক্ষরে লেখা এসেছে `CONFLICT (content): Merge conflict in...`। তুমি বুঝতে পারছ না এখন কী করবে।
**সহজ সমাধান:** মার্জ করা ক্যান্সেল করে দাও এবং আগের নিরাপদ অবস্থায় চলে যাও।
```bash
git merge --abort
```
> [!NOTE]
> এটা দিলে তোমার মার্জ ওখানেই বাতিল হয়ে যাবে এবং তোমার ফাইলগুলো ঠিক মার্জ শুরু করার আগের অবস্থায় ফেরত যাবে। এরপর তুমি ঠান্ডা মাথায় টিমমেটের সাথে কথা বলে আবার ট্রাই করতে পারো। 

---

### 🚨 ভুল ৬: গিটহাবে ভুল কোড Push করে দিয়েছি! (বিপজ্জনক)
**সমস্যা:** তুমি ভুলে কোনো সিক্রেট পাসওয়ার্ড বা ভুল কোড `main` ব্রাঞ্চে পুশ করে দিয়েছ। এখন সেটা গিটহাবে সবাই দেখতে পাচ্ছে।
**সহজ সমাধান:** আনডু (Undo) করে নতুন পুশ দাও।

প্রথমে চেক করো কোন কমিটে ভুল করেছ:
```bash
git log --oneline
```
ধরো তোমার ভুল কমিটের আইডি `7a4bd3`। এখন সেটাকে বাদ দিয়ে আগের অবস্থায় যেতে হবে:
```bash
git revert 7a4bd3
git push origin main
```
> [!TIP]
> `revert` দিলে তোমার গিটহাবে কোনো হিস্ট্রি ডিলিট হয় না, বরং গিট অটোমেটিকালি তোমার ভুলগুলোকে কেটে ঠিক করে নতুন একটা "ফিক্সড" কমিট অ্যাড করে দেয়। এটাই সবচেয়ে নিরাপদ উপায়।

---

### 🎯 সামারি: ৩টি সুপার পাওয়ার টুল
1. **`git status`** : তুমি কোথায় আছো, কী করছ — বুঝতে না পারলে সবসময় এটা দাও।
2. **`git log`** : তোমার প্রজেক্টের হিস্ট্রি দেখার জন্য। 
3. **`git stash`** : সাময়িক সময়ের জন্য কাজ লুকিয়ে রাখার জন্য। 

এই গাইডটা ফলো করলে তুমি গিট হাবের যেকোনো কনফিউশন অনায়াসে পার করে যেতে পারবে! 🚀
